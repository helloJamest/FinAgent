## Problem Statement

A股短线交易核心逻辑：每日收盘后获取游资龙虎榜和连板天梯数据，遍历热门板块，
重点筛选1进2/2进3/3进4的连板股票，输出板块策略和买入建议。

## What Makes This Cool

将市场情绪、资金流向、板块热度三个维度合并为一份结构化打板策略报告，
短线投资者一目了然：哪个板块值得关注、哪些股票值得打板、具体怎么操作。

## Constraints

- 复用现有 screener 系统架构（strategy YAML + LLM screener + validator + pipeline + service）
- 数据源使用 AKShare 专用 API（龙虎榜、涨停池、涨停概念）
- 复用现有 ScreenerCategory = MOMENTUM
- 遵循 AGENTS.md 目录边界
- 不顺手修改无关代码

## Premises

1. 龙虎榜数据包含游资买卖明细，是短线情绪的重要信号 — agree
2. 连板天梯能自动识别1进2/2进3/3进4的晋级股 — agree
3. 涨停概念/板块数据能反映当日最热的板块 — agree
4. LLM 可以基于这些结构化数据生成可读性强的策略分析 — agree
5. 所有数据源免费且无需额外 API key — agree

## Approaches Considered

### Approach A: 完整打板流水线
- 新建 BoardDataFetcher + BoardScreener 模块
- 新增 `screen_board_play` 策略 YAML
- 在 ScreenerCategory 中注册新策略
- 新增 API endpoint `/api/v1/screener/board`
- Effort: M, Risk: Med, Reuses: screener 框架、data_provider 模式

### Approach B: 纯 LLM 驱动
- 只新增一个策略 YAML，prompt 里引导 LLM 调用数据工具
- 不新增数据获取模块
- Effort: S, Risk: High (数据质量不稳定)

### Approach C: 扩展现有 screen_monster_ambush
- 在已有妖股策略上叠加龙虎榜和连板逻辑
- Effort: S, Risk: Med (耦合度高，不利于扩展)

## Recommended Approach

Approach A — 完整打板流水线。模块化设计，每个层次职责清晰，
复用现有 screener 框架，扩展性好。

## Success Criteria

1. 能正确获取当日龙虎榜、涨停池、连板天梯数据
2. 能自动识别 1进2/2进3/3进4 晋级股
3. 能输出板块策略 + 连板预测 + 买入策略
4. 注册到现有 screener 系统，可通过 CLI 和 API 调用
